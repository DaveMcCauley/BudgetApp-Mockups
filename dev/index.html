1.3 Typography

BudgetApp uses Lato exclusively. The only exception is when a monospaced font is required. The Lato typeface is imported in the _fonts.scss partial file. Although it should always be available, the _fonts.scss includes alternates for fallback.

  • For normal purposes use the $font-normal-stack definition.
  • For monospaced type use the $font-mono-stack definition.

1.3.1 Typefaces and Weights

There are four type weights available for use with the normal font stack. For the monospaced font stack ONLY the normal weight should be used.

1.3.2 Hierarchy

Try to use the predefined text sizes. This gives a consistent visual hierachy across the application. If you find it necessary to change from the standard text sizes, ensure you size the text in em units, so that the text resizes in proportion to the standard sizes. PX sized text should be avoided unless absolutely necessary.

1.4 Iconography

BudgetApp ustilizes Bootstrap 3 as a base framework. Bootstrap comes with a large set of glyphs pre-installed. In most cases attempt to find a glyph that fits your needs from these icons. A handy tool for locating Bootstrap icons can be found over at Cheatsheets.com

You may also use Google's Material icons by including the _material-icons.scss partial in a page specific sass file. Icons and insertion codes can be found over at design.google.com

Create new icons only as a last resort (for now.)

2.0 Components

2.1 Buttons

Our buttons are created with standard bootstrap construction. For example:

	
		<button type="button" class="btn btn-default btn-sm"> Preferences </button>
		<button type="button" class="btn btn-default btn-sm"> Ok </button>
		<button type="button" class="btn btn-default btn-sm"> Schedule </button>

2.1.1 Button Color

There are six basic button colors.

		
			<button type="button" class="btn btn-default btn-sm">Default</button>
			<button type="button" class="btn btn-primary btn-sm">Primary</button>
			<button type="button" class="btn btn-success btn-sm">Success</button>
			<button type="button" class="btn btn-info btn-sm">Info</button>
			<button type="button" class="btn btn-warning btn-sm">Warning</button>
			<button type="button" class="btn btn-danger btn-sm">Danger</button>

2.1.2 Button Size

BudgetApp buttons are sized with .btn-sm by default. Only use alternate sizes of the bootstrap default (no included class) or large (.btn-lg). Never use the extra small button class (.btn-xs).

2.1.2 Buttons with Glyphs

Buttons with glyphs provide a fast way of picking choices. Use these in places where the user is presented an option, or many options to aid in quickly recognizing the deisred action. 

	
		<button type="button" class="btn btn-default btn-sm">
  		<span class="glyphicon glyphicon-cog" aria-hidden="true"></span> Preferences
		</button>
		<button type="button" class="btn btn-default btn-sm">
  		<span class="glyphicon glyphicon-ok" aria-hidden="true"></span> Ok
		</button>
		<button type="button" class="btn btn-default btn-sm">
  		<span class="glyphicon glyphicon-time" aria-hidden="true"></span> Schedule
		</button>

Button can also use glyphs without any text, such as:

	
		<button type="button" class="btn btn-default" aria-label="Left Align">
  		<span class="glyphicon glyphicon-align-left" aria-hidden="true"></span>
		</button>
		<button type="button" class="btn btn-default" aria-label="Left Align">
		  <span class="glyphicon glyphicon-align-center" aria-hidden="true"></span>
		</button>
		<button type="button" class="btn btn-default" aria-label="Left Align">
		  <span class="glyphicon glyphicon-align-right" aria-hidden="true"></span>
		</button>

2.1.3 Button Groups

Buttons can may grouped when they share a common useage. Basic sample is to wrap the buttons with .bth-group. Note that the btn-group-sm attribute is only requred for the group, and not each button. 

	
		<div class="btn-group btn-group-sm" role="group" aria-label="...">
			<button type="button" class="btn btn-default ">Left</button>
			<button type="button" class="btn btn-default">Middle</button>
			<button type="button" class="btn btn-default">Right</button>
		</div>
Tooltips and popovers in button groups require special setting When using tooltips or popovers on elements within a .btn-group, you'll have to specify the option container: 'body' to avoid unwanted side effects (such as the element growing wider and/or losing its rounded corners when the tooltip or popover is triggered).
